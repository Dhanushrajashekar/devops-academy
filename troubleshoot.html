🔧 Troubleshooting Guide

Real errors, real fixes. Searchable by error message, tool, and symptom. Covers Docker, Kubernetes, CI/CD, and AWS.

🐳 Docker Errors
Cannot connect to Docker daemon Docker daemon not running or permission denied
DockerEasy
›
Symptoms
Cannot connect to the Docker daemon at unix:///var/run/docker.sock. Is the docker daemon running?
permission denied while trying to connect to the Docker daemon socket
1
Check if Docker is running
sudo systemctl status docker

If inactive/dead: sudo systemctl start docker && sudo systemctl enable docker

2
Add your user to the docker group (permission denied fix)
sudo usermod -aG docker $USER newgrp docker # Apply without logout

Then verify: docker run hello-world

3
On macOS — Docker Desktop not started

Open Docker Desktop from Applications. The whale icon in the menu bar must be running before any docker commands work.

no space left on device Docker disk space exhausted — images, containers, volumes
DockerEasy
›
Symptoms
write /var/lib/docker/overlay2/...: no space left on device
failed to write to /dev/stdout: no space left on device
1
Check disk usage
df -h /var/lib/docker docker system df # Docker-specific breakdown
2
Safe cleanup — removes dangling resources only
docker system prune # Remove stopped containers + dangling images docker image prune -a # Remove ALL unused images (more aggressive) docker volume prune # Remove unused volumes
3
Nuclear option — full reset
docker system prune -a --volumes # ⚠️ Removes everything not running

On Docker Desktop: Settings → Troubleshoot → Clean / Purge data

💡
Prevent This in CI
Add docker system prune -af as a post-build step in your CI pipeline. CI runners accumulate images fast across many builds.
port is already allocated Port conflict — another process using the same port
DockerEasy
›
Symptoms
Bind for 0.0.0.0:8080 failed: port is already allocated
Error starting userland proxy: listen tcp 0.0.0.0:80: bind: address already in use
1
Find what's using the port
lsof -i :8080 # macOS/Linux netstat -tulpn | grep 8080 # Linux # Windows: netstat -ano | findstr :8080
2
Option A — Use a different host port
docker run -p 8081:8080 my-image # Map host:8081 → container:8080
3
Option B — Stop the conflicting container
docker ps # Find container using the port docker stop <container-id>
COPY failed: file not found Dockerfile COPY/ADD — file not in build context
DockerMedium
›
Symptoms
COPY failed: file not found in build context or excluded by .dockerignore: stat app.jar: file does not exist
1
Understand build context

The build context is the directory you pass to docker build. Only files IN that directory (and not in .dockerignore) are accessible to COPY. Paths in COPY are relative to the build context root, not your current directory.

2
Check your build command
docker build -t my-app . # Context = current directory docker build -t my-app -f Dockerfile.prod ./src # Context = ./src
3
Check .dockerignore isn't excluding the file
cat .dockerignore # Make sure your file isn't listed here
4
Common multi-stage build mistake
# Wrong — can't COPY from outside build context COPY ../other-service/lib.jar /app/ # Right — use multi-stage or restructure build context FROM maven:3.9 AS build COPY . . RUN mvn package FROM openjdk:21 COPY --from=build /app/target/app.jar /app/
exec format error Wrong architecture — building on M1/M2 Mac for AMD64 Linux
DockerMedium
›
Symptoms
standard_init_linux.go: exec user process caused "exec format error"
WARNING: The requested image's platform (linux/amd64) does not match the detected host platform (linux/arm64/v8)
1
Build for the target platform explicitly
# Build for Linux AMD64 (what EKS nodes run) docker build --platform linux/amd64 -t my-app:latest .
2
Build multi-platform image (supports both)
docker buildx build \ --platform linux/amd64,linux/arm64 \ -t my-registry/my-app:latest \ --push .
3
In GitHub Actions — always specify platform
- name: Build and push uses: docker/build-push-action@v5 with: platforms: linux/amd64 # Explicit for EKS (x86 nodes) push: true tags: ${{ env.ECR_REGISTRY }}/my-app:${{ github.sha }}
☸️ Kubernetes Errors
CrashLoopBackOff Pod keeps crashing and restarting — diagnostic workflow
K8sMedium
›
Symptoms
STATUS: CrashLoopBackOff RESTARTS: 5 (Last: 2m ago)
Back-off restarting failed container
1
Get the exit code and last logs
kubectl describe pod <pod-name> # Check Events and Exit Code kubectl logs <pod-name> --previous # Logs from the crashed container

Exit Code 1 = app error. Exit Code 137 = OOMKilled. Exit Code 139 = segfault.

2
OOMKilled (Exit 137) — increase memory limit
resources: requests: memory: "256Mi" limits: memory: "512Mi" # Increase this if OOMKilled
3
App crashes on start — test image locally
docker run --rm -it my-image:tag sh # Does the container start? docker run --rm my-image:tag # Does it run and stay up?
4
Missing env var or secret — check container env
kubectl exec -it <pod-name> -- env | sort # List all env vars in running pod kubectl get secret <secret-name> -o yaml # Check secret exists
ImagePullBackOff Kubernetes can't pull the container image
K8sAWSEasy
›
Symptoms
Failed to pull image "123456.dkr.ecr.us-east-1.amazonaws.com/app:latest": ... unauthorized
STATUS: ImagePullBackOff | ErrImagePull
1
Verify the image exists and tag is correct
aws ecr describe-images --repository-name my-app --region us-east-1 # Check: does the tag you're deploying actually exist?
2
EKS — check node IAM role has ECR permissions
# Node role must have this policy attached: arn:aws:iam::aws:policy/AmazonEC2ContainerRegistryReadOnly # Check current node role kubectl get nodes -o wide aws iam list-attached-role-policies --role-name <node-role-name>
3
Private registry — check imagePullSecret
# Create secret from Docker config kubectl create secret docker-registry regcred \ --docker-server=<registry> \ --docker-username=<user> \ --docker-password=<token> # Add to Pod spec imagePullSecrets: - name: regcred
Pending Pod stuck in Pending — can't be scheduled
K8sMedium
›
Symptoms
STATUS: Pending (no RESTARTS, stays there indefinitely)
0/3 nodes are available: 3 Insufficient cpu
0/3 nodes are available: 3 node(s) didn't match pod's node affinity
1
Read the scheduler message
kubectl describe pod <pod-name> | grep -A 20 Events

The Events section tells you exactly WHY scheduling failed.

2
Insufficient CPU/Memory — check node capacity
kubectl describe nodes | grep -A 5 "Allocated resources" kubectl top nodes # requires metrics-server

Fix: scale node group, reduce pod resource requests, or add nodes.

3
Taint/Toleration mismatch
kubectl get nodes -o custom-columns=NAME:.metadata.name,TAINTS:.spec.taints # Add toleration to pod spec if node has taint tolerations: - key: "dedicated" operator: "Equal" value: "gpu" effect: "NoSchedule"
4
PVC not bound — storage class or zone issue
kubectl get pvc # Status should be Bound, not Pending kubectl describe pvc <name> # Check Events for provisioner errors
OOMKilled Container killed — out of memory
K8sMedium
›
Symptoms
Last State: Terminated Reason: OOMKilled Exit Code: 137
STATUS: CrashLoopBackOff with increasing restart count
1
Confirm it's OOMKilled
kubectl describe pod <pod-name> | grep -A 5 "Last State" # Should show: Reason: OOMKilled, Exit Code: 137
2
Check actual memory usage before setting limits
kubectl top pods --containers -n <namespace> kubectl top nodes
3
Increase memory limit (rule of thumb: 2x average usage)
resources: requests: memory: "256Mi" # Scheduler uses this for placement limits: memory: "512Mi" # OOM kill threshold — set 25-50% above peak
4
JVM apps — set heap size relative to container limit
env: - name: JAVA_OPTS value: "-Xmx400m -Xms256m" # Xmx must be < memory limit

Java 17+ has container-aware defaults — use -XX:MaxRAMPercentage=75.0 instead of fixed Xmx.

connection refused Service not routing traffic — selector/endpoint mismatch
K8sHard
›
Symptoms
curl: (7) Failed to connect to my-svc.default.svc.cluster.local port 80: Connection refused
upstream connect error or disconnect/reset before headers
1
Check if Service has Endpoints
kubectl get endpoints my-svc # Should show pod IPs — if <none>, the selector matches nothing
2
Verify selector matches pod labels exactly
kubectl get svc my-svc -o yaml | grep -A 5 selector kubectl get pods --show-labels | grep <app-name> # Label in Service selector MUST match label on pod — case-sensitive
3
Verify pod is Ready (not just Running)
kubectl get pods # READY column must show 1/1 not 0/1 # Pods only enter Endpoints when readinessProbe passes
4
Test connectivity from inside the cluster
kubectl run debug --image=busybox --rm -it -- sh # Inside the pod: wget -qO- http://my-svc.default.svc.cluster.local:80
Unauthorized / Forbidden kubectl access denied — RBAC or kubeconfig issue
K8sHard
›
Symptoms
Error from server (Forbidden): pods is forbidden: User "jane" cannot list resource "pods"
error: You must be logged in to the server (Unauthorized)
Unable to connect to the server: x509: certificate signed by unknown authority
1
Check current context
kubectl config current-context # Are you in the right cluster? kubectl config get-contexts # List all contexts kubectl config use-context prod # Switch context
2
Refresh EKS kubeconfig credentials
aws eks update-kubeconfig --region us-east-1 --name my-cluster # This refreshes your auth token (EKS tokens expire after 15 minutes)
3
RBAC Forbidden — check role bindings
kubectl auth can-i list pods --as=jane -n production kubectl get rolebindings,clusterrolebindings --all-namespaces | grep jane # Grant access: kubectl create rolebinding jane-viewer \ --clusterrole=view \ --user=jane \ --namespace=production
⚙️ CI/CD Errors
Not authorized to perform sts:AssumeRoleWithWebIdentity GitHub Actions can't assume AWS IAM role (OIDC)
CI/CDAWSHard
›
Symptoms
Error: Not authorized to perform sts:AssumeRoleWithWebIdentity
Error: Could not load credentials from any providers
1
Verify workflow has OIDC permissions
permissions: id-token: write # ← MUST have this contents: read
2
Check IAM role trust policy
{ "Principal": { "Federated": "arn:aws:iam::ACCOUNT:oidc-provider/token.actions.githubusercontent.com" }, "Condition": { "StringEquals": { "token.actions.githubusercontent.com:sub": "repo:ORG/REPO:ref:refs/heads/main" } } } # The sub condition must match your repo and branch EXACTLY
3
Verify OIDC provider exists in IAM
aws iam list-open-id-connect-providers # Must include: token.actions.githubusercontent.com
denied: Your authorization token has expired Docker push to ECR fails — authentication token expired
CI/CDAWSEasy
›
Symptoms
denied: Your authorization token has expired. Reauthenticate and try again.
no basic auth credentials
1
ECR login tokens expire after 12 hours — re-authenticate
# In GitHub Actions — use the official action - name: Login to Amazon ECR uses: aws-actions/amazon-ecr-login@v2 # This handles auth automatically using the assumed AWS role
2
Manual login (if doing it manually)
aws ecr get-login-password --region us-east-1 | \ docker login --username AWS --password-stdin \ $AWS_ACCOUNT_ID.dkr.ecr.us-east-1.amazonaws.com # Must run this BEFORE every docker push, not just once per job
Build taking 10+ minutes CI pipeline is too slow — caching strategies
CI/CDMedium
›
Symptoms
Every run installs all dependencies from scratch (5-8 min for npm install)
Docker image build ignores layer cache — rebuilds everything
1
Cache npm/pip/maven dependencies in GitHub Actions
- uses: actions/cache@v4 with: path: ~/.npm key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }} restore-keys: ${{ runner.os }}-node-
2
Optimize Dockerfile layer order for cache hits
# BAD — copies code before installing deps, cache busts every change COPY . . RUN npm install # GOOD — install deps first (cached until package.json changes) COPY package*.json ./ RUN npm ci COPY . .
3
Use Docker layer cache with GitHub Actions
- uses: docker/build-push-action@v5 with: cache-from: type=gha cache-to: type=gha,mode=max push: true tags: ${{ env.IMAGE_TAG }}
☁️ AWS Errors
CREATE_FAILED EKS cluster creation fails or times out
AWSHard
›
Symptoms
CREATE_FAILED: The maximum number of VPCs has been reached
eksctl create cluster: timed out waiting for CloudFormation stack
No output in eksctl after 10+ minutes
1
Check CloudFormation stack for the actual error
aws cloudformation describe-stack-events \ --stack-name eksctl-my-cluster-cluster \ --query 'StackEvents[?ResourceStatus==`CREATE_FAILED`]'
2
VPC limit hit — request increase or delete old VPCs
aws ec2 describe-vpcs --query 'length(Vpcs)' # Default limit is 5 # Request limit increase: AWS Console → Service Quotas → VPCs per Region
3
Clean up failed stack before retrying
eksctl delete cluster --name my-cluster --region us-east-1 # OR via CloudFormation console — delete the ROLLBACK_COMPLETE stacks
Error acquiring the state lock Terraform state locked — DynamoDB lock stuck
AWSCI/CDMedium
›
Symptoms
Error acquiring the state lock: ConditionalCheckFailedException: The conditional request failed
Lock Info: ID: abc123, Who: CI Runner, Created: 2h ago
1
Verify the lock is truly stale (no apply in progress)

Check if a CI job is currently running. Check CloudTrail for recent Terraform activity. Only force-unlock if you're 100% sure no apply is running — concurrent applies can corrupt state.

2
Force unlock with the Lock ID
terraform force-unlock <LOCK-ID> # The Lock ID is shown in the error message
3
Manual DynamoDB cleanup (if terraform command fails)
aws dynamodb delete-item \ --table-name terraform-state-lock \ --key '{"LockID": {"S": "my-bucket/path/to/terraform.tfstate-md5"}}'
Ingress ADDRESS pending Kubernetes Ingress stuck with no ALB address assigned
K8sAWSHard
›
Symptoms
kubectl get ingress → ADDRESS column is blank after 5+ minutes
No ALB appearing in AWS Console after applying Ingress resource
1
Check if AWS Load Balancer Controller is running
kubectl get pods -n kube-system | grep aws-load-balancer kubectl logs -n kube-system -l app.kubernetes.io/name=aws-load-balancer-controller
2
Check Ingress class annotation
# Must have correct annotation metadata: annotations: kubernetes.io/ingress.class: alb alb.ingress.kubernetes.io/scheme: internet-facing # or internal alb.ingress.kubernetes.io/target-type: ip # Required for EKS
3
Check controller IAM permissions
# Controller needs these IAM permissions (usually via IRSA) elasticloadbalancing:CreateLoadBalancer elasticloadbalancing:DescribeLoadBalancers ec2:DescribeSubnets # Check controller's service account annotation: kubectl describe sa aws-load-balancer-controller -n kube-system | grep eks.amazonaws.com
4
Tag subnets for ALB discovery
# Public subnets for internet-facing ALB must have tag: kubernetes.io/role/elb = 1 # Private subnets for internal ALB: kubernetes.io/role/internal-elb = 1 # Also tag with cluster name: kubernetes.io/cluster/my-cluster = shared

Quick Reference — Diagnostic Commands

Run these first when something goes wrong.

☸️ Kubernetes — First 5 Commands When Things Break

kubectl get pods -AList all pods in all namespaces + status
kubectl describe pod <name>Events, resource requests, container state — start here
kubectl logs <pod> --previousLogs from the last crashed container
kubectl get events --sort-by=.lastTimestampCluster-wide events sorted by time
kubectl rollout status deploy/<name>Is a rollout stuck? Shows progress
kubectl rollout undo deploy/<name>Roll back to previous version immediately
kubectl top pods --containersActual CPU/memory usage per container
kubectl exec -it <pod> -- bashShell into a running pod for live debug
kubectl get endpoints <svc>Is the Service backed by healthy pods?
kubectl auth can-i <verb> <resource>Test RBAC permissions for current user

🐳 Docker — Diagnostic Commands

docker ps -aAll containers including stopped ones
docker logs <container> --tail 50Last 50 lines of container output
docker inspect <container>Full container config — env vars, mounts, network
docker system dfDisk usage breakdown by images/containers/volumes
docker exec -it <container> shShell into running container
docker statsLive CPU/memory/network usage per container
docker history <image>Layer sizes — find bloated layers
docker system prune -afRemove all unused images/containers (CI cleanup)

☁️ AWS CLI — Diagnostic Commands

aws sts get-caller-identityWho am I? — verify IAM credentials are working
aws eks update-kubeconfig --name <cluster>Refresh kubectl credentials for EKS
aws ecr describe-repositoriesList ECR repos in current region
aws ec2 describe-instances --query 'Reservations[].Instances[].State'Check instance states
aws cloudformation describe-stack-events --stack-name <name>Why did my stack fail?
aws logs tail <log-group> --followStream CloudWatch logs in terminal
