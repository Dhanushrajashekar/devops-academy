File System & Navigation

The Linux file system is a tree starting at /. Everything is a file — even devices and network sockets.

Navigation Commands

pwd # Print Working Directory — where am I? ls # List files ls -la # Long format + hidden files (files starting with .) ls -lh # Human-readable file sizes (KB, MB, GB) ls -lt # Sort by modification time (newest first) cd /etc # Change to absolute path cd ../logs # Relative — go up one, then into logs cd ~ # Go to your home directory cd - # Go back to previous directory # Finding files find / -name "*.conf" 2>/dev/null # Find all .conf files find /var/log -mtime -1 # Modified in last 24 hours find /home -size +100M # Files larger than 100MB locate nginx.conf # Fast search (uses DB — run updatedb first)

File Operations

# Create touch file.txt # Create empty file / update timestamp mkdir -p dir/sub/sub2 # Create nested directories # Copy, Move, Delete cp file.txt backup.txt # Copy file cp -r dir/ dir_backup/ # Copy directory recursively mv old.txt new.txt # Rename or move rm file.txt # Delete file rm -rf directory/ # Delete directory recursively — NO confirmation! # View files cat file.txt # Print entire file less file.txt # Paginated view (q to quit, / to search) head -20 file.txt # First 20 lines tail -50 file.txt # Last 50 lines tail -f /var/log/syslog # Follow log in real-time (Ctrl+C to stop) # Links ln -s /etc/nginx/nginx.conf ~/nginx.conf # Symlink (like a shortcut)
🎯
Interview: What is the difference between a hard link and a soft link?
Hard link: Points to the same inode (data blocks on disk). Both the original and link must be on the same filesystem. Deleting the original doesn't break the hard link.

Soft link (symlink): Points to the original file's path. If the original is deleted, the symlink breaks. Works across filesystems. Used for: version aliasing (python → python3.11), shared config files.

Permissions & Ownership

Linux permissions control who can read, write, and execute every file. This comes up in every Linux interview.

Critical~25 min

Reading Permission Strings

$ ls -la /etc/passwd -rw-r--r-- 1 root root 2847 Jan 15 10:23 /etc/passwd │└┬┘└┬┘└┬┘ │ │ │ └── Other (everyone else): r-- = read only │ │ └───── Group (root group): r-- = read only │ └──────── Owner (root user): rw- = read + write └────────── File type: - = regular file, d = directory, l = symlink

Run as program

chmod — Change Permissions

# Numeric (octal) mode — add up r+w+x for each group chmod 755 script.sh # rwxr-xr-x — owner: all, group+others: read+execute chmod 644 file.txt # rw-r--r-- — owner: read+write, group+others: read chmod 600 id_rsa # rw------- — owner only (required for SSH private keys!) chmod 777 file # rwxrwxrwx — everyone everything (NEVER in production) # Symbolic mode — easier to understand chmod u+x script.sh # Add execute for user (owner) chmod g-w file.txt # Remove write from group chmod o=r file.txt # Set others to read-only exactly chmod a+r file.txt # Add read for all (user+group+others) chmod -R 755 /var/www # Recursive — apply to all files in directory

chown — Change Ownership

chown alice file.txt # Change owner to alice chown alice:developers file.txt # Change owner AND group chown -R www-data:www-data /var/www/html # Recursive (web server files) chgrp developers project/ # Change group only

Common Permission Patterns

OctalStringUse Case
755rwxr-xr-xScripts, directories, web root — owner can modify, others can read/execute
644rw-r--r--Config files, web files — owner edits, others read
600rw-------SSH private keys, secret files — owner only
700rwx------Private scripts — owner only
400r--------Read-only sensitive files (SSL certs, API keys)
⚠️
SSH will refuse your private key if permissions are wrong
WARNING: UNPROTECTED PRIVATE KEY FILE!
SSH requires private keys to be 600 (owner read/write only). Fix: chmod 600 ~/.ssh/id_rsa. This is one of the most common Linux interview gotchas.

sudo — Running as Root

sudo command # Run single command as root sudo -i # Switch to root shell (use sparingly) sudo -u alice command # Run as a different user sudo !! # Re-run last command with sudo # Who can use sudo? Check /etc/sudoers sudo visudo # Edit sudoers safely (never edit directly) # Example sudoers entry: alice ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart nginx # alice can restart nginx without a password, on any host

Text Processing — grep, awk, sed

The Swiss Army knife of Linux. Parse logs, transform files, extract data — without opening a text editor.

Core~30 min

grep — Search

grep "ERROR" app.log # Lines containing ERROR grep -i "error" app.log # Case insensitive grep -r "TODO" /src/ # Recursive through directory grep -n "ERROR" app.log # Show line numbers grep -v "DEBUG" app.log # Exclude DEBUG lines grep -A 3 "FATAL" app.log # Show 3 lines After match grep -B 2 "FATAL" app.log # Show 2 lines Before match grep -C 5 "FATAL" app.log # Show 5 lines Context (before+after) grep -E "ERROR|WARN" app.log # Extended regex — match ERROR or WARN grep -c "ERROR" app.log # Count matches (not lines with output)

sed — Stream Editor (Find & Replace)

# Basic substitution: s/find/replace/flags sed 's/foo/bar/' file.txt # Replace first occurrence per line sed 's/foo/bar/g' file.txt # Replace ALL occurrences (global) sed 's/foo/bar/gi' file.txt # Case insensitive global replace sed -i 's/localhost/prod.db/g' app.conf # Edit file IN-PLACE (⚠️ no undo) sed -i.bak 's/old/new/g' file.txt # In-place + keep .bak backup # Delete lines sed '/^#/d' config.conf # Delete comment lines (starting with #) sed '/^$/d' file.txt # Delete blank lines sed '5d' file.txt # Delete line 5 # Print specific lines sed -n '10,20p' file.txt # Print lines 10–20

awk — Column Processing

# awk sees each line as fields: $1 $2 $3... $NF = last field awk '{print $1}' file.txt # Print first column awk '{print $1, $3}' file.txt # Print 1st and 3rd columns awk -F: '{print $1}' /etc/passwd # Use : as delimiter — print usernames awk '{print NR, $0}' file.txt # Add line numbers (NR = Number of Row) # Filtering awk '$3 > 100' data.txt # Print lines where 3rd field > 100 awk '/ERROR/ {print $0}' app.log # Print lines matching ERROR awk 'NR==5' file.txt # Print only line 5 # Real-world: parse nginx access log awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -10 # → Top 10 client IPs by request count awk '$9 == 500' /var/log/nginx/access.log # → All requests that returned HTTP 500
🎯
Interview: "Show me only the ERROR lines from this log file, with 2 lines of context"
grep -C 2 "ERROR" /var/log/app.log

Then they might follow up: "Now count how many errors per hour." Answer: grep "ERROR" app.log | awk '{print $2}' | cut -d: -f1 | sort | uniq -c (extracts the hour from timestamps and counts by hour).

Process Management

View, control, and troubleshoot running processes. Essential for production debugging.

Critical~25 min

Viewing Processes

ps aux # All processes: USER PID %CPU %MEM COMMAND ps aux | grep nginx # Find nginx process ps -ef --forest # Tree view showing parent-child relationships top # Live process monitor (q to quit) htop # Better top — colour, mouse support (install if missing) # top keyboard shortcuts: # P = sort by CPU | M = sort by memory | k = kill process | q = quit

Signals — Killing Processes

SIGTERM (15)
Graceful Stop
Ask process to stop cleanly. Default for kill. Process can handle cleanup.
SIGKILL (9)
Force Kill
Kernel kills immediately. Cannot be caught or ignored. Last resort only.
SIGHUP (1)
Reload Config
Reload config without restart. Used by nginx, sshd, cron.
SIGINT (2)
Interrupt
What Ctrl+C sends. Interrupt the running process.
kill 1234 # Send SIGTERM to PID 1234 (graceful) kill -9 1234 # Send SIGKILL (force — only if SIGTERM fails) kill -HUP 1234 # Reload config (nginx: reload without dropping connections) killall nginx # Kill all processes named nginx pkill -f "python app" # Kill by matching command string # Find PID of a process pgrep nginx # Print PID(s) of nginx pidof sshd # Get PID of sshd

Background Jobs

command & # Run in background jobs # List background jobs fg %1 # Bring job 1 to foreground bg %1 # Resume job 1 in background Ctrl+Z # Suspend foreground process nohup command & # Run even after logout (ignores SIGHUP) # Better: use screen or tmux for persistent sessions tmux new -s mysession # Create named tmux session tmux attach -t mysession # Re-attach after logout
🎯
Interview: What's the difference between kill -9 and kill -15?
SIGTERM (15, default): Politely asks the process to stop. The process can catch this signal, save state, close connections, and shut down cleanly. This is correct in production.

SIGKILL (9): The kernel forcefully terminates the process immediately — the process cannot intercept this. Files may be left unclosed, connections broken mid-stream. Use only when SIGTERM fails after waiting 30+ seconds.

Users, Groups & sudo

Linux multi-user system management — who can do what.

Core~20 min

Key Files

/etc/passwd # User accounts: username:x:UID:GID:comment:home:shell /etc/shadow # Hashed passwords (root-only readable) /etc/group # Groups: groupname:x:GID:members /etc/sudoers # Who can use sudo and how # Read /etc/passwd cat /etc/passwd | cut -d: -f1 # List all usernames id alice # Show UID, GID, groups for alice whoami # Current username who # Who is logged in right now last # Login history

User Management

# Create user useradd -m -s /bin/bash alice # -m: create home dir, -s: set shell passwd alice # Set password # Modify user usermod -aG sudo alice # Add alice to sudo group (-a = append) usermod -aG docker alice # Add to docker group usermod -s /bin/bash alice # Change shell usermod -L alice # Lock account (disable login) usermod -U alice # Unlock account # Delete user userdel alice # Delete user (keep home dir) userdel -r alice # Delete user AND home directory # Groups groupadd developers # Create group groups alice # Show alice's groups

Switch Users

su - alice # Switch to alice's environment (full login shell) su alice # Switch to alice (keeps current env) sudo su - # Become root (if you have sudo access) exit # Return to previous user

SSH & Remote Access

SSH is how you manage every Linux server remotely. Know it inside out.

Critical~25 min

SSH Basics

ssh user@server # Connect (uses password or default key) ssh -i ~/.ssh/mykey.pem ec2-user@1.2.3.4 # Specify private key ssh -p 2222 user@server # Non-standard port ssh -L 8080:localhost:80 user@server # Local port forward # Exit SSH session exit # or Ctrl+D

SSH Key Authentication (No Password)

# Step 1: Generate key pair on your local machine ssh-keygen -t ed25519 -C "my-laptop" # Creates: ~/.ssh/id_ed25519 (private) + ~/.ssh/id_ed25519.pub (public) # Step 2: Copy public key to server ssh-copy-id user@server # OR manually append to server's ~/.ssh/authorized_keys # Step 3: Set correct permissions chmod 700 ~/.ssh chmod 600 ~/.ssh/id_ed25519 chmod 644 ~/.ssh/id_ed25519.pub chmod 600 ~/.ssh/authorized_keys # On the server

SSH Config File — Stop Typing Long Commands

# ~/.ssh/config Host prod-web HostName 10.0.1.50 User ec2-user IdentityFile ~/.ssh/prod.pem Port 22 Host bastion HostName 52.10.20.30 User ubuntu IdentityFile ~/.ssh/bastion.pem Host prod-db HostName 10.0.3.100 User postgres IdentityFile ~/.ssh/prod.pem ProxyJump bastion # Connect via bastion host (jump host) # Now just type: ssh prod-web ssh prod-db # Automatically goes through bastion

Hardening SSH (/etc/ssh/sshd_config)

PermitRootLogin no # Never allow root login directly PasswordAuthentication no # Key-only auth (after setting up keys!) Port 2222 # Non-standard port (reduces noise, not real security) AllowUsers alice bob # Whitelist specific users MaxAuthTries 3 # Limit brute force attempts # After editing: sudo sshd -t # Test config before reloading! sudo systemctl reload sshd

SCP & rsync — File Transfer

scp file.txt user@server:/home/user/ # Copy TO server scp user@server:/var/log/app.log ./ # Copy FROM server scp -r ./dir user@server:/opt/ # Copy directory rsync -avz ./src/ user@server:/opt/app/ # Sync directory (only changed files) rsync -avz --delete ./src/ user@server:/opt/ # Mirror (deletes files not in src)

Bash Scripting

Automate repetitive tasks. Interviewers often ask you to write a script on the spot.

Core~35 min

Script Structure

#!/bin/bash # ↑ Shebang — tells OS to use bash to run this script set -e # Exit immediately on any error set -u # Treat unset variables as errors set -o pipefail # Catch errors in pipes echo "Script started at $(date)"

Variables & Input

NAME="Alice" # Variable (no spaces around =) echo "Hello $NAME" # Use variable echo "Hello ${NAME}s" # Use variable with text after (curly braces) # Command-line arguments $0 # Script name $1 # First argument $2 # Second argument $# # Number of arguments $@ # All arguments # Command substitution TODAY=$(date +%Y-%m-%d) DISK_USAGE=$(df -h / | tail -1 | awk '{print $5}') echo "Disk: $DISK_USAGE" # Read user input read -p "Enter username: " USERNAME

Conditionals

# If / elif / else if [ "$1" == "start" ]; then echo "Starting..." elif [ "$1" == "stop" ]; then echo "Stopping..." else echo "Usage: $0 start|stop" exit 1 fi # Common test conditions [ -f /etc/nginx.conf ] # File exists [ -d /var/log ] # Directory exists [ -z "$VAR" ] # Variable is empty [ -n "$VAR" ] # Variable is not empty [ "$A" == "$B" ] # String equality [ $NUM -gt 10 ] # Number greater than

Loops

# For loop for server in web1 web2 web3; do echo "Deploying to $server..." ssh $server "sudo systemctl restart app" done # For loop over files for file in /var/log/*.log; do echo "Processing: $file" gzip "$file" done # While loop COUNT=0 while [ $COUNT -lt 5 ]; do echo "Attempt $COUNT" COUNT=$((COUNT + 1)) done

Real-World Script Example

#!/bin/bash set -euo pipefail # Deployment script APP_NAME="myapp" DEPLOY_DIR="/opt/$APP_NAME" LOG_FILE="/var/log/${APP_NAME}-deploy.log" log() { echo "[$(date '+%H:%M:%S')] $1" | tee -a "$LOG_FILE"; } log "Starting deployment of $APP_NAME" # Check if running as root if [ "$(id -u)" != "0" ]; then log "ERROR: Must run as root" exit 1 fi # Backup current version if [ -d "$DEPLOY_DIR" ]; then log "Backing up current version..." cp -r "$DEPLOY_DIR" "${DEPLOY_DIR}.bak.$(date +%Y%m%d%H%M%S)" fi # Deploy log "Deploying new version..." systemctl stop "$APP_NAME" || true rsync -avz /tmp/release/ "$DEPLOY_DIR/" systemctl start "$APP_NAME" # Health check sleep 5 if systemctl is-active --quiet "$APP_NAME"; then log "Deployment successful!" else log "ERROR: Service failed to start. Rolling back..." cp -r "${DEPLOY_DIR}.bak."* "$DEPLOY_DIR" systemctl start "$APP_NAME" exit 1 fi

Cron & Systemd Services

Schedule jobs and manage long-running services — the backbone of Linux automation.

Core~20 min

Cron — Scheduled Jobs

crontab -e # Edit your crontab (opens in $EDITOR) crontab -l # List current crontab entries crontab -r # Remove crontab (⚠️ no confirmation!) # Cron syntax: minute hour day month weekday command # * * * * * # │ │ │ │ └── Day of week (0=Sun, 6=Sat) # │ │ │ └──────── Month (1-12) # │ │ └───────────── Day of month (1-31) # │ └────────────────── Hour (0-23) # └─────────────────────── Minute (0-59) 0 2 * * * /opt/backup.sh # Daily at 2:00 AM */5 * * * * /opt/healthcheck.sh # Every 5 minutes 0 9 * * 1-5 /opt/report.sh # Weekdays at 9 AM 0 0 1 * * /opt/monthly.sh # 1st of every month @reboot /opt/startup.sh # Run once on boot

Systemd — Service Management

# Control services systemctl start nginx # Start now systemctl stop nginx # Stop now systemctl restart nginx # Stop + start systemctl reload nginx # Reload config without stopping (if supported) systemctl enable nginx # Start automatically on boot systemctl disable nginx # Don't start on boot systemctl status nginx # Current status + recent logs systemctl is-active nginx # Returns 0 if active (for scripts) systemctl list-units --type=service # List all services # Create your own service: /etc/systemd/system/myapp.service [Unit] Description=My Application After=network.target [Service] Type=simple User=myapp WorkingDirectory=/opt/myapp ExecStart=/opt/myapp/bin/server Restart=on-failure RestartSec=5 [Install] WantedBy=multi-user.target # Load and start systemctl daemon-reload systemctl enable --now myapp
